Docker Experiment – 10
Running and Managing Docker Containers
Aim

To understand Docker containerization and run/manage containers using Docker commands.

Procedure
Step 1: Check Docker Installation
docker --help
docker version

Displays available Docker commands and installed version.

Step 2: Run First Container (Ubuntu)
docker container run -it ubuntu top

Explanation:

run → creates and starts container
-it → interactive terminal
ubuntu → image name
top → command executed inside container

If image is not available locally, Docker downloads it automatically from Docker Hub.

Step 3: View Running Containers

Open another terminal and run:

docker container ls

Shows running containers.

Step 4: Enter Running Container
docker container exec -it <container_id> bash

Example:

docker container exec -it b3ad2a23fab3 bash

Now you are inside container.

Check running processes:

ps -ef

Exit container:

exit
Step 5: Run Multiple Containers
Run Nginx Container
docker container run -d -p 8080:80 --name nginx nginx
-d → detached mode
-p 8080:80 → maps host port 8080 to container port 80

Check output:

curl localhost:8080
Run MongoDB Container
docker container run -d -p 8081:27017 --name mongo mongo:4.4

Check output:

curl localhost:8081
Step 6: List All Running Containers
docker container ls
Step 7: Stop Containers
docker container stop nginx mongo
Step 8: Remove Stopped Containers
docker system prune -f

Removes:

stopped containers
unused networks
dangling images
Step 9: Create Dockerfile
FROM ubuntu
RUN apt-get update
CMD ["echo","Hello from Docker Container"]
Step 10: Build Image
docker build -t myimage .
Step 11: Run Custom Image
docker run myimage

Expected Output:

Hello from Docker Container
Important Docker Commands
Command	Purpose
docker ps	Show running containers
docker images	Show images
docker stop <id>	Stop container
docker rm <id>	Remove container
docker exec -it <id> bash	Enter container
docker build -t name .	Build image
Result

Docker containers were successfully created, managed, and executed using Docker CLI commands.
