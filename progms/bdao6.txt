Experiment – 8

Deploy Big Data Workloads using Cloud-Managed Services. Launch and Configure a Cluster using Amazon EMR, Azure HDInsight, or Google Cloud Dataproc________________________________________
Aim
To deploy and execute Big Data workloads by launching and configuring a cluster using Amazon Web Services EMR, Microsoft Azure HDInsight, or Google Cloud Dataproc.

Tools Required
•	Cloud Account:
o	Amazon EMR
o	Azure HDInsight
o	Google Cloud Dataproc
•	Internet Browser
•	Dataset (sample text file / log file)
•	Storage Service
o	Amazon S3
o	Azure Blob Storage
o	Google Cloud Storage

Cloud-Managed Big Data Services

Cloud providers offer managed platforms for Big Data processing. These services simplify cluster deployment, configuration, and management.

Features
•	Automatic cluster provisioning
•	Scalability
•	Integrated storage
•	Managed frameworks (Hadoop, Spark, Hive)
•	Cost optimization

Common Frameworks
•	Apache Hadoop
•	Apache Spark
•	Apache Hive

Option 1: Deploy Big Data Workload using Amazon EMR

Step 1: Login to AWS Console
1.	Open AWS Management Console.
2.	Navigate to Services → Analytics → EMR.

Step 2: Create Cluster
1.	Click Create Cluster.
2.	Select Quick Options.
Configure:

Parameter	Value
Cluster Name	BigDataLabCluster
Release Version	Latest
Applications	Hadoop, Spark
Instance Type	m5.xlarge
Number of Instances	3

Step 3: Configure Storage
1.	Create a bucket in Amazon S3.
2.	Upload a sample dataset (e.g., sample.txt).

Step 4: Launch Cluster
1.	Review configuration.
2.	Click Create Cluster.
3.	Wait until cluster status becomes Running.
Step 5: Run a Sample Big Data Job
Run a Spark Word Count job.
Example command:
spark-submit --class org.apache.spark.examples.JavaWordCount \
/usr/lib/spark/examples/jars/spark-examples.jar \
s3://bucket-name/sample.txt
Step 6: Monitor Job
1.	Go to Steps tab in EMR cluster.
2.	Check job status.
Step 7: View Output
Output will be stored in the S3 bucket.
Option 2: Deploy Big Data Workload using Azure HDInsight
Step 1: Login to Azure Portal
Open **Microsoft Azure Portal.
Step 2: Create Cluster
1.	Go to Create Resource
2.	Select HDInsight Cluster
Configure:

Parameter		Value
Cluster Type		Spark
Cluster Name		bigdatalabcluster
Resource Group	BigDataLab
Storage		Azure Blob
Step 3: Upload Dataset
Upload sample dataset to Azure Blob Storage.
Step 4: Submit Spark Job
Example command:
spark-submit wordcount.py sample.txt
Step 5: Monitor Job
Use YARN Resource Manager UI.
Option 3: Deploy Big Data Workload using Google Cloud Dataproc
Step 1: Login to Console
Open Google Cloud Console.
Step 2: Create Dataproc Cluster
Navigate to Dataproc → Clusters → Create Cluster
Configuration:
Parameter	Value
Cluster Name	bigdata-cluster
Region	us-central1
Master Node	1
Worker Nodes	2
Step 3: Upload Dataset
Upload file to Google Cloud Storage.
Step 4: Submit Job
Example Spark job:
gcloud dataproc jobs submit spark \
--cluster bigdata-cluster \
--class org.apache.spark.examples.JavaWordCount \
--jars file:///usr/lib/spark/examples/jars/spark-examples.jar \
gs://bucket/sample.txt
Step 5: Monitor Job
Check job status in Dataproc Job Dashboard.
Sample Dataset
Example text file:
Cloud computing enables scalable big data processing.
Big data analytics provides insights from large datasets.
Cloud platforms simplify cluster management.
